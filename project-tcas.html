Senior Project

KUSE ADMISSION

เว็บแอปพลิเคชันให้ข้อมูลการรับเข้าศึกษา คณะวิทยาศาสตร์และวิศวกรรมศาสตร์

โปรเจกต์จบ (Senior Project) พัฒนาในช่วงชั้นปีที่ 3–4 มหาวิทยาลัยเกษตรศาสตร์ วิทยาเขตเฉลิมพระเกียรติ จังหวัดสกลนคร (KUSE)

พัฒนาขึ้นเพื่อเป็นศูนย์กลางรวบรวมข้อมูลข่าวสารและประกาศรับสมัครเข้าศึกษาต่อของคณะวิทยาศาสตร์และวิศวกรรมศาสตร์ มก.ฉกส. ให้อยู่ในที่เดียว ค้นหาง่าย เป็นระบบ และเข้าถึงข้อมูลที่ถูกต้องทันสมัยได้สะดวกยิ่งขึ้น


Problem

แม้คณะฯ จะมีช่องทางเผยแพร่ข้อมูล เช่น เว็บไซต์และเพจ Facebook แต่ข้อมูลการรับเข้าศึกษายังคงกระจัดกระจายและไม่เป็นระบบ ผู้สนใจต้องใช้เวลานานในการค้นหาประกาศย้อนหลัง โดยเฉพาะข้อมูลที่มีเฉพาะบนโซเชียลมีเดียซึ่งถูกโพสต์ใหม่มาแทนที่อยู่เสมอ ทำให้พลาดข้อมูลสำคัญได้ง่าย

Solution

พัฒนาเว็บแอปพลิเคชัน “KUSE แอดมิดชัน” ให้เป็นศูนย์กลางข้อมูลการรับเข้าศึกษาของคณะโดยเฉพาะ พร้อมระบบค้นหาและกรองข้อมูล (Search & Filter) ระบบสถานะประกาศ (Status Label) และระบบดาวน์โหลดเอกสารรับสมัคร เพื่อให้ผู้สนใจเข้าถึงข้อมูลที่ถูกต้อง ทันสมัย และตรวจสอบได้ง่าย

My Role

  • ออกแบบ UX/UI ของระบบทั้งหมดด้วย Figma
  • วิเคราะห์ User Flow และออกแบบโครงสร้างของระบบ
  • พัฒนาส่วนหน้าบ้าน (Front-end) ด้วย HTML, CSS, JavaScript และ React
  • ออกแบบโครงสร้างฐานข้อมูลและระบบจัดการประกาศรับสมัคร (TCAS แต่ละรอบ)

สถานะโปรเจกต์ : อยู่ระหว่างการพัฒนา (In Progress)

ขอบเขตของระบบ

ฝั่งผู้ใช้งานทั่วไป (Front-end)

  • ดูข้อมูลการรับเข้าศึกษาและหลักสูตรที่เปิดสอนของคณะ
  • ดูประกาศรับสมัครปัจจุบันและย้อนหลัง พร้อมดาวน์โหลดไฟล์ PDF
  • ค้นหาประกาศผ่าน Search Box และ Dropdown Filter
  • ดูสถานะประกาศแบบ Real-time ผ่าน Status Label
รอเปิดรับ กำลังเปิดรับ ปิดรับแล้ว

ฝั่งผู้ดูแลระบบ (Back-end)

  • เข้าสู่ระบบและจัดการบัญชีผู้ดูแลระบบ
  • จัดการประกาศรับสมัครในแต่ละรอบของระบบ TCAS (รอบ 1–4)
  • จัดการไฟล์เอกสารประกาศและระยะเวลาการรับสมัคร
  • จัดการข้อมูลหลักสูตรและรูปภาพแบนเนอร์ประชาสัมพันธ์

หลักสูตรที่เปิดสอนในระบบ

วิศวกรรมโยธา วิศวกรรมไฟฟ้า วิศวกรรมเครื่องกลและการผลิต วิศวกรรมอุตสาหการ วิศวกรรมคอมพิวเตอร์ วิศวกรรมสิ่งแวดล้อมเพื่อการพัฒนาอย่างยั่งยืน วิทยาการคอมพิวเตอร์ เคมีประยุกต์

Process

ออกแบบโดยใช้แนวคิด User-Centered Design — คลิกที่รูปเพื่อดูขนาดเต็ม

1

User Persona

ภาพตัวแทนกลุ่มผู้ใช้งานหลักของระบบ ( นักเรียนที่สนใจสมัครเรียน) ใช้วิเคราะห์พฤติกรรม เป้าหมาย และ Pain Point ก่อนเริ่มออกแบบระบบ

2

Wireframe

โครงร่างเค้าโครงหน้าจอ (Layout) แบบคร่าว ๆ ที่ใช้วางตำแหน่งองค์ประกอบของแต่ละหน้า ก่อนนำไปออกแบบ UI และพัฒนาจริง

3

User Journey Map

แผนภาพแสดงเส้นทางการใช้งานของผู้ใช้ตั้งแต่ต้นจนจบ เพื่อวิเคราะห์ขั้นตอนและ Pain Point ในแต่ละจุดสัมผัส (Touchpoint)

Result

  • ผู้ใช้งานสามารถค้นหาและเข้าถึงข้อมูลการรับเข้าศึกษาได้ง่ายและรวดเร็วขึ้น
  • ลดความกระจัดกระจายของข้อมูลที่เคยอยู่หลายแพลตฟอร์ม
  • เพิ่มประสิทธิภาพในการประชาสัมพันธ์ของคณะให้เป็นระเบียบมากขึ้น
  • รองรับการตรวจสอบประกาศรับสมัครย้อนหลังได้อย่างเป็นระบบ
